# TotalRecall Admin UI

Static TypeScript UI for prompt orchestration, runtime provider credentials,
Mem0 and provider monitoring, catalogue review, memory deletion, learning
review, and feature flag visibility.

The UI calls only backend HTTP APIs:

- `GET /v1/catalogue`
- `GET /v1/catalogue/{entity_id}`
- `POST /v1/generations`
- `GET /v1/credentials`
- `PUT /v1/credentials/{credential_key}`
- `DELETE /v1/credentials/{credential_key}`
- `GET /v1/monitoring/summary`
- `GET /v1/monitoring/memory`
- `GET /v1/monitoring/providers`
- `GET /v1/monitoring/token-efficiency`
- `DELETE /v1/memories/{entity_id}`
- `GET /v1/learning/runs`
- `POST /v1/learning/runs`
- `POST /v1/learning/runs/{run_id}/approve/{discovery_id}`
- `POST /v1/learning/runs/{run_id}/reject/{discovery_id}`
- `GET /v1/flags`
- `GET /v1/metrics`

## Connection

When the Docker stack is running, use either API base:

```text
http://localhost:4173/v1
http://localhost:8000/v1
```

Use the local development bearer token:

```text
dev-token
```

`localhost:4173` is allowed by the API CORS defaults. If you serve the UI from a
different origin, add that origin to `TOTALRECALL_CORS_ALLOWED_ORIGINS`.

## Monitoring

The Monitoring view shows:

- Overall API health
- Active and configured memory adapter
- Mem0 credential, SDK, write, and fail-open status
- Memory search/get/upsert/delete counters and latency averages
- Provider registration, credential readiness, and health
- Latest context-planning token-efficiency values

## Learning Path Warnings

The Learning view displays run-level warnings returned by the API. If the API
runs in Docker, submitted Windows paths such as
`C:\ENV\test-env-management\tests` must be mounted into the container or mapped
to a mounted path:

```bash
TOTALRECALL_LEARNING_WORKSPACE_HOST=C:\ENV
TOTALRECALL_LEARNING_PATH_MAPPINGS={"C:\\ENV":"/learning-workspace"}
```

After that, submit either the original host path or
`/learning-workspace/test-env-management/tests`.

## Mem0 Setup

Use the `Credentials` view `Mem0 Setup` panel to configure TotalRecall memory:

1. Paste the Mem0 API key generated by the hosted Mem0 dashboard or by the
   self-hosted dashboard setup/bootstrap flow.
2. For self-hosted Mem0, enter the REST API host reachable from the TotalRecall
   API process. Use `http://localhost:8888` for local Python, or
   `http://host.docker.internal:8888` when TotalRecall runs in Docker and Mem0
   runs on the host. Leave it blank for hosted Mem0.
3. Keep `Activate Mem0 for TotalRecall memory after saving` checked.
4. Click `Configure Mem0`.

The UI saves `mem0_api_key`, optionally saves `mem0_host`, and activates
`memory.adapter=mem0_v1`, `memory.write_enabled=true`, and
`memory.fail_open_on_search=true`. Open `Monitoring` after saving to confirm
Mem0 is active and healthy.

When TotalRecall is started with the repository-level `docker-compose.mem0.yml`
overlay, the API and memory wrapper are preconfigured for self-hosted Mem0:

```bash
docker compose -f docker-compose.yml -f docker-compose.mem0.yml up --build app memory-wrapper admin-ui mem0
```

Set `OPENAI_API_KEY`, `MEM0_ADMIN_API_KEY`, and `MEM0_JWT_SECRET` before
starting the overlay. TotalRecall reaches Mem0 through Docker at
`http://mem0:8000`; the host can inspect the Mem0 REST API at
`http://localhost:8888`.

The `Self-hosted Mem0 Docker` panel can also submit those three values to the
API, write `local-secrets/mem0-selfhost.env`, activate the `mem0_v1` memory
adapter, and start `mem0` plus `mem0-postgres`. API-side Docker startup is
blocked unless the backend was started with:

```bash
TOTALRECALL_ADMIN_DOCKER_CONTROL_ENABLED=true
```

Use this only on trusted local developer machines where the API process has
Docker Compose access to the repository root. The submitted secrets are stored
locally and are not returned to the browser after saving.

The repository `docker-compose.mem0.yml` overlay builds the Mem0 API image from
the official Mem0 source tarball by default. This avoids Docker Hub
platform-manifest failures for `mem0/mem0-api-server:latest` on local
`linux/amd64` Docker engines. The overlay also mounts a `mem0-history-data`
volume at `/app/history` so the Mem0 SQLite history database has a writable,
persistent directory.

Open `index.html` directly or serve the directory:

```bash
python -m http.server 4173
```

Build from TypeScript when the frontend toolchain is installed:

```bash
npm install
npm run build
```

The checked-in `dist/app.js` keeps the UI runnable in Python-only development
and CI environments.
